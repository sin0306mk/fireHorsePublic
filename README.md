# システム概要書

**多摩区 火災情報 自動通知システム（FireHorse）**

> このドキュメントは、システムを管理・説明する立場の方（管理者・自治会役員等）を対象としています。
> 通知を受け取る側の方は [business-spec.md](business-spec.md) をご覧ください。
> 技術的な仕様の詳細は [../spec/architecture.md](../spec/architecture.md) を参照してください。

---

## 1. このシステムが解決すること

川崎市消防局の火災情報ウェブサイトは、人が手動で見に行かないと情報を得られません。このシステムは **そのページを1分ごとに自動確認し、多摩区で火災が起きたときにLINEとメールで関係者へ自動通知** します。

## 2. 火災検知から通知までの流れ

```
┌─────────────────────────────────────────────────────────────────┐
│                     1分ごとに自動で繰り返す                      │
│                                                                 │
│  ① 川崎市消防局のサイトを確認                                    │
│     https://sc.city.kawasaki.jp/saigai/index.htm               │
│                    ↓                                            │
│  ② ページが前回から更新されていなければ → そのまま終了           │
│     （更新されていれば次へ）                                     │
│                    ↓                                            │
│  ③ 多摩区の火災情報かどうかを判定                               │
│     ・高津区・麻生区・生田地区は除外                            │
│     ・火災・鎮火に関する情報のみ対象                            │
│                    ↓                                            │
│  ④ 同じ情報を2回通知しないようチェック（重複排除）              │
│                    ↓                                            │
│  ⑤ 通知を送信                                                   │
│     ・火災発生 → LINE ＋ メール                                  │
│     ・鎮火・対応完了 → メールのみ                               │
└─────────────────────────────────────────────────────────────────┘
```

## 3. どの地域の・どんな情報が・どこに届くか

### 通知エリアの全体像

```
川崎市全域
  ├── 高津区 ................ 通知なし（対象外）
  ├── 麻生区 ................ 通知なし（対象外）
  └── 多摩区
        ├── 生田地区 ......... 通知なし（生田消防署管轄のため除外）
        │    （生田・枡形・三田・栗谷・長沢）
        │
        └── 多摩区その他 ..... メール通知あり ✉
              ├── 宿河原（1〜7丁目）... LINE通知もあり 📱✉
              ├── 堰（1〜3丁目）  ... LINE通知もあり 📱✉
              ├── 長尾（1〜7丁目）
              ├── 登戸新町
              ├── 登戸
              ├── 和泉
              ├── 菅地区（各町）
              ├── 寺尾台
              ├── 中野島
              ├── 布田
              └── 上記以外の多摩区全域
```

> **LINEは宿河原・堰のみ**
> 現在の設定では、LINE通知は **宿河原** と **堰** エリアの火災発生時のみ送信されます。その他の多摩区エリアはメールのみです。
> この設定は `config.yaml` の `line_target_areas` で変更できます。

---

### 状況別・通知先の違い

| 状況 | LINE（宿河原・堰のみ） | メール（多摩区全体） |
|---|---|---|
| 🔥 **火災の通報**（出火） | ✅ 届く | ✅ 届く |
| 💧 **鎮火** | ❌ 届かない | ✅ 届く |
| ✅ **対応は終了しました** | ❌ 届かない | ✅ 届く |
| その他（ガス漏れ・災害等） | ❌ 届かない | ❌ 届かない |

---

### LINEとメールの内容の違い

**LINE に届く内容（火災発生時のみ）**

消防局サイトに掲載された文章をそのまま転送します。

```
３月１５日　１２時２４分　頃　多摩区　宿河原３丁目　付近で発生した　火災の通報
```

シンプルな1行テキストです。

---

**メールに届く内容（火災発生・鎮火・対応完了すべて）**

件名と本文に整形した詳細情報が届きます。

```
件名: 【火災】多摩区 宿河原３丁目:2026年03月15日 13時11分現在 １２時２４分
      ↑ 状況に応じて [鎮火] / [完了] に変わります

本文:
  【 詳 細 情 報 】
  2026年03月15日 13時11分現在

    発生時刻: ３月１５日 １２時２４分
    発生場所: 多摩区 宿河原３丁目
    状況    : 火災の通報

  http://www.mapion.co.jp/address/14135/4:3/  ← 現場周辺の地図リンク

   詳細な情報は消防署（宿河原出張所）に直接電話してください。
   0449000119
```

---

### 一枚で見るまとめ

```
火災発生（宿河原・堰）
  → LINE：1行テキスト（速報）
  → メール：詳細情報＋地図リンク

火災発生（宿河原・堰 以外の多摩区）
  → LINE：届かない
  → メール：詳細情報＋地図リンク

鎮火・対応完了（多摩区全体）
  → LINE：届かない
  → メール：詳細情報＋地図リンク（件名が [鎮火] / [完了] に変わる）
```

## 5. システムを構成するAWSのサービス

このシステムは Amazon Web Services（AWS）上で動いています。サーバーを自前で用意・管理する必要がなく、使った分だけ課金されるクラウドサービスです。

```
┌──────────────────────────────────────────────────────────────┐
│  AWS クラウド（東京リージョン）                               │
│                                                              │
│  ┌────────────────────┐                                      │
│  │ EventBridge        │  ← 「1分ごとに起動する」タイマー     │
│  │ Scheduler          │                                      │
│  └─────────┬──────────┘                                      │
│            │ 1分ごとに起動                                   │
│  ┌─────────▼──────────┐   ┌──────────────────────────────┐   │
│  │ Lambda             │◀──│ Secrets Manager              │   │
│  │（処理プログラム）   │   │（LINEトークン等の機密情報を  │   │
│  │                    │   │ 安全に保管）                  │   │
│  └──┬──────────┬──────┘   └──────────────────────────────┘   │
│     │          │                                             │
│  ┌──▼──────┐   │                                             │
│  │DynamoDB │   │ 通知送信                                    │
│  │（通知済 │   │                                             │
│  │ 履歴の  │   └──────────────┬──────────────────────────┐   │
│  │ 記録）  │                  │                          │   │
│  └─────────┘           ┌─────▼──────┐          ┌────────▼─┐ │
│                        │ SES        │          │ LINE     │ │
│  ┌──────────────────┐  │（メール    │          │ Platform │ │
│  │ CloudWatch       │  │ 送信）     │          │（AWS外）  │ │
│  │（ログ・監視）    │  └────────────┘          └──────────┘ │
│  └──────────────────┘                                       │
└──────────────────────────────────────────────────────────────┘
```

### 各サービスの役割（一覧）

| サービス名 | 役割 | たとえると |
|---|---|---|
| **EventBridge Scheduler** | 1分ごとにプログラムを自動起動するタイマー | 目覚まし時計 |
| **Lambda** | 実際の処理を行うプログラムの実行環境 | プログラムが動くコンピューター（使うときだけ起動） |
| **DynamoDB** | 通知済み火災情報の履歴を記録するデータベース | 「この火災はもう通知した」のメモ帳 |
| **Secrets Manager** | LINEトークンなど機密情報を安全に保管する金庫 | 鍵のかかった金庫 |
| **SES** | メール送信サービス | 郵便局（メール専用） |
| **SQS（DLQ）** | プログラムが失敗したときのメッセージを一時保持するバッファ | 再配達ボックス |
| **SNS** | エラー発生時のアラートをメールで転送する中継サービス | 緊急連絡網 |
| **CloudWatch** | プログラムの動作ログ収集・エラー監視・アラート管理 | 監視カメラ＋警報機 |
| **IAM** | 各サービスのアクセス権限管理 | 入館証の発行・管理 |

## 6. 信頼性・監視のしくみ

### 自動エラー検知

以下の異常が発生すると、CloudWatch がアラートを発報し管理者にメールで通知します。

| 異常の種類 | 条件 | 意味 |
|---|---|---|
| Lambda エラー | 5分以内に1回以上のエラー | プログラムが正常に動いていない |
| DLQ 蓄積 | 1件以上のメッセージが滞留 | 再試行が尽きた失敗処理が保持されている（要調査） |

### 重複通知の防止

同じ火災情報を2回以上通知しないよう、DynamoDB に通知済み情報を記録しています。記録は **7日後に自動削除** されます。

### ページ変化がない場合の早期終了

川崎市消防局のサイトが前回から更新されていない場合、即座に処理を終了します（通常 1〜2秒で完了）。

### 同時実行の制限

プログラムの同時実行数を1に制限しており、二重起動による重複通知を防いでいます。

## 8. 管理者が行う主な作業

### 設定変更が必要な場面

「デプロイ」とは、設定ファイルの変更内容をAWS上のシステムに反映させる作業です（`make deploy` コマンドを実行します）。

| 場面 | 変更先 | 方法 |
|---|---|---|
| メール受信者を追加・変更する | `deploy/.env` の `EMAIL_RECIPIENTS` | カンマ区切りで記載 → **デプロイ** |
| LINE通知エリアを変更する | `src/app/config.yaml` の `line_target_areas` | エリア名を追記 → **デプロイ** |
| LINEトークンを更新する | AWS Secrets Manager | AWS コンソールから直接更新（**デプロイ不要**） |
| 新しい通知エリアを追加する | `src/app/config.yaml` の `place_mappings` | エリア定義を追記 → **デプロイ** |

### 日常監視コマンド

AWS 認証情報 CSV（`../` に配置）と `deploy/.env` があれば、Docker 経由で以下を実行できます。AWS CLI のローカルインストールは不要です。

```bash
# 全項目を一括確認（推奨）
make status

# 個別確認
make ddb          # DynamoDB: ページ最終タイムスタンプ・通知済み件数
make logs         # Lambda: 直近20件の INFO/ERROR ログ
make logs FILTER=ERROR  # エラーログのみ表示
make line         # LINE: 今月の送信数・上限・友達数
```

## 9. EC2方式からの移行について

2026年3月以前は EC2（仮想サーバー）上で動いていましたが、以下の理由でサーバーレス（Lambda）方式に移行しました。

| 比較項目 | 旧方式（EC2） | 現在（Lambda） |
|---|---|---|
| サーバー管理 | 必要（OS更新・監視） | 不要 |
| 費用 | 常時起動のため固定費が高い | 処理時のみ課金（大幅削減） |
| 状態管理 | ローカルファイル（サーバー障害でデータ消失リスク） | DynamoDB（障害に強い） |
| Python バージョン | Python 2 | Python 3.12 |
| 可用性 | サーバー障害で停止 | AWS が管理するため高可用性 |
